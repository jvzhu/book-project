# Contributing

Thank you for helping improve this book project.

## Ways to contribute

- **Propose edits**: open a pull request with concrete content improvements.
- **Submit feedback**: open a feedback issue with clarity, accuracy, or structure comments.
- **Suggest content**: open a content suggestion issue for missing topics or chapter ideas.

## Proposing edits (Pull Requests)

1. Fork the repository and create a branch for your change.
2. Keep changes focused and explain the rationale clearly.
3. If you edit chapter content, update the related outline or section headings when needed.
4. Open a pull request with:
   - a short summary of what changed,
   - why the change helps readers,
   - any follow-up ideas.

## Submitting feedback (Issues)

Use the **Feedback** issue template and include:
- where the issue appears (chapter/section),
- what was unclear or incorrect,
- a suggested improvement (if available).

## Suggesting new content (Issues)

Use the **Content Suggestion** issue template and include:
- the proposed topic,
- target audience/value,
- which chapter it belongs to (or if new),
- optional references.

## Collaboration guidelines

- Be respectful and constructive.
- Prefer small, reviewable contributions.
- Keep discussions actionable and reader-focused.
